# Contributing Guidelines

Thank you for your interest in contributing to **Software Supply Chain Security – Assignment 1**!  
We appreciate contributions that help improve the project, fix bugs, enhance documentation, or strengthen supply-chain security practices.

---

## Pull Request Process

To contribute code or documentation changes, please follow these steps:

1. **Fork the repository** 
   Click the **Fork** button on the GitHub page to create your own copy of the repo.

2. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name

--

## Issue Reporting

If you encounter bugs, security issues, or have feature suggestions, please report them through GitHub Issues:

Check if the issue already exists in the Issues tab

If not found, open a new issue with the following details:

	Title: Clear and concise summary of the issue

	Description: Steps to reproduce, expected vs. actual behavior

	Environment: OS, runtime version, and relevant configurations

	Logs or screenshots (if applicable)

--

## Code Style and Testing Guidelines

To maintain consistency and reliability across contributions, please follow these guidelines:

Code Style

	Follow the conventions of the project’s language (e.g., PEP 8 for Python, Prettier/ESLint for JavaScript).

	Use clear variable and function names.

	Write comments for complex logic or configurations.

	Keep commits focused — one logical change per commit.

Testing

	Ensure all existing tests pass before submitting a PR.

	Add new tests for new functionality or bug fixes.
	
	Example
		# For Python
		pytest

		# For Node.js
		npm test

Linting and Static Analysis

	Run linting tools before submitting:

		flake8 .        # Python example
		eslint .        # Node.js example


	Address any warnings or errors before creating a PR.

Contributor Recognition

	All accepted contributors will be credited in the project’s release notes and GitHub Contributors list.
	We value your time and effort in improving the project!
